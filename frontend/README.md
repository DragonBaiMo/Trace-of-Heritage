# 寻戏之旅——经典戏曲文化分享互动管理系统的设计与实现前端

## 概述
本前端基于 Vite + Vue 3 + Pinia + Vue Router 构建，提供登录、资源列表与资源提交等核心闭环能力，以便与 FastAPI 后端联动完成“提交-审核-查看”流程。同时新增资源仪表盘、分页检索、最新动态与管理员审计日志等看板能力。

## 环境准备
1. 安装 Node.js (>=18)。
2. 复制 `.env.example` 为 `.env` 并按需调整 `VITE_API_BASE`。

## 常用命令
- `npm install`：安装依赖。
- `npm run dev`：启动开发调试服务。
- `npm run build`：构建生产版本到 `dist/` 目录。

## 目录结构
```
frontend/
├── index.html
├── package.json
├── src
│   ├── api        # Axios 封装与领域接口
│   ├── router     # 路由与守卫
│   ├── store      # Pinia 状态管理
│   ├── styles     # 全局样式
│   └── views      # 业务页面
└── vite.config.ts
```

## 与后端的联调
启动后端服务 (`uvicorn app.main:app --reload`)，再执行 `npm run dev`。使用默认管理员账户登录后，可完成资源提交并在资源库页面查看状态。

## UI 设计焕新方案
为提升界面观感与一致性，建议按以下思路重新设计：

1. **全局布局**：采用三栏响应式结构，左侧为固定导航（渐变背景 + 模块图标），中部内容区使用卡片式容器、右侧保留活动/公告/排行榜组件，宽屏下并排展示，窄屏自动折叠为抽屉。
2. **色彩与字体**：主色调选用「丹砂红 + 靛青」对比，搭配米白色背景；标题使用思源黑体半粗，正文保持常规字重，并在暗色模式下切换为深灰背景、暖白文字。
3. **组件体系**：
   - 卡片投影统一为 0 2px 20px rgba(0,0,0,0.08)，边角 12px 圆角；
   - 表单控件添加浮动标签与实时校验提示，错误信息使用红色提示条；
   - 列表与表格加入分段标题、标签颜色区分状态（草稿灰、待审橙、通过绿、驳回红）。
4. **互动反馈**：按钮 hover 增加微动效（scale 1.02）与渐变描边，重要操作前弹出中文确认对话框；加载状态统一使用骨架屏或线性进度条。
5. **可视化**：仪表盘采用柔和面积图 + 柱状图组合，图例和筛选控件固定在卡片顶栏；地图或轨迹信息使用暖色系渐层折线，重点节点配以徽章。
6. **可达性**：对比度遵循 WCAG AA 标准，所有交互控件提供键盘焦点样式；文案统一改为中文描述，避免英文缩写干扰。

按照上述方案实施，可显著降低问号标记带来的困惑，让界面在保持信息密度的同时更具沉浸感与文化底蕴。
